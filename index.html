Styleguide

General guidelines for Chambers Judd web projects

View My GitHub Profile

This is an overview of the tools and techniques we use project to project. It will change over time and we're not married to it so think of it as guidelines rather than gospel.

General

  • Assets should be an assets folder in the root, i.e. /assets/css /assets/js /assets/img
  • When editing code, use spaces not tabs
  • If in doubt, write a comment - assume you won't be there when someone else opens the file

CSS

  • Written using SASS in SMACCS structure.
  • Comments should be written using KSS syntax
  • Comments should be ruby (i.e. // not /* )
  • Use Bourbon mixins
  • Use classes not ID's wherever possible
  • Give JS only classes a js- prefix

JS

  • All JS should be tested against JS Lint. There is a Sublime Text plugin which let's you do this as you go.
  • Minify JS prior to production (or on the fly with a Sublime Text Build Task) using Uglify JS
  • Write code in modules. CSS Tricks module pattern is a good example.

Images

  • Use sprites where possible - Spritecow is a useful tool for this
  • Minify large images using imageoptim prior to production

Git

Always pull a repository before you start working to check it's up to date

  • Please work on branches for a specific feature, then when it's complete merge into the 'staging' branch (as opposed to master). Name branches in the style navigation_feature. Overview of the idea at http://blog.teamtreehouse.com/our-simple-git-workflow

  • Create tags for releases. These should be preceded by the date and include a description e.g. 20120101-site-launch

Databases

  • Databases should be named with the tech as a prefix, then the project shortname (e.g. for an expressionengine site it would be 'ee_projectname')
  • Database exports should be gzipped where possible and exported to a _database folder in the root of the project
  • Exported DB dumps should be prefixed by the date and include the initials of the person working on it - e.g. 20120101-JC-ee_projectname.sql.gz
